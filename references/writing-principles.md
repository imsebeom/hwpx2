# 보고서 작성 원칙 (Writing Principles)

> 출처: 약 20여 년의 공공기관 근무 경력을 가진 직원의 「업무용 글쓰기」 강의자료(전 25페이지)
> + 사례강좌 자료(양식의 중요성 / 문장 줄이기 / 적의를 보이는 것들)
> + 공공기관 사무관리지침 관행

이 문서는 SKILL.md에서 참조되는 **글쓰기 품질 기준의 단일 소스**입니다.
Claude는 4단계 워크플로우 중 **③ 콘텐츠 매핑**과 **④ 레이아웃 최적화** 단계에서 이 문서를 점검 기준으로 사용합니다.

---

## 1. 업무용 글쓰기의 본질

> **업무의 본질 = 문제해결.**
> 따라서 업무용 글쓰기의 목적도 문제해결이며,
> 보고서의 구조 = 문제해결 프로세스 그 자체.

업무용 글의 기본 3요소:

| 요소 | 정체 | 점검 질문 |
|------|------|----------|
| **구성** (논리적 요소) | 자료 배치 + 프로세스 적용 | 왜 → 어떻게 → 무엇을 결정? 흐름이 보이는가? |
| **양식** (시각적 요소) | 가독성·전달력 위한 문서 규격 | 익숙한 양식인가, 표준에서 벗어나지 않는가? |
| **표현** (언어적 요소) | 독자 고려한 단어·문장 | 간결한가, 친절한가, 개조식인가? |

스킬은 이 3요소 모두를 자동 점검 대상으로 삼습니다.

## 2. 보고서 논리 패턴 (Fishing → Reasoning → Message)

공공기관 보고서 작성 강의자료(p.9) 핵심 구성:

```
시작(Why)        → 왜 이 사업/보고를 하는가?           [제목 · 개요 · 추진배경]
중간(How)        → 어떻게 해결할 것인가?                [현황(주요내용) · 문제점/원인 · 해결방안]
마무리(What)     → 무엇을 결정/판단할 것인가?           [기대효과 · 조치사항 · 추진계획]
```

각 구성항목의 역할:

- **제목** — 독자의 관심·흥미 유발 (해결방안 + 기대효과를 압축)
- **개요** — 용건, 핵심 메시지 (3줄 이내, 두괄식)
- **추진배경** — 계기·조건·경과 / 취지·이유·필요성
- **현황** — 과제로 주어진 상황 (목표와 현실의 차이를 표출)
- **문제점·원인** — 그 차이가 발생한 부정적 요소와 원인
- **해결방안** — 구체성과 실현가능성이 핵심
- **기대효과** — 의사결정을 이끌어내기 위한 결과 예측
- **조치사항** — 추진주체·예산·일정·역할분담·규정·장애요인·극복방안

> ⚠️ AI가 콘텐츠를 매핑할 때 **이 항목 중 빠진 게 있으면** `※ AI 보완 필요` 플래그를 붙여 사용자에게 경고해야 함.

## 3. 개조식 문체 (가장 중요한 표현 원칙)

> **가독성이 높은 개조식 문체로 간결하게 기술하는 것이 원칙이며, 독자를 감안하여 적절한 단어와 문장 서술구조를 택한다.**
> — 강의자료 p.12

### 3-1. 개조식의 정의

- **개요**: 글을 쓸 때 앞에 번호·기호를 붙여가며 짧게 끊어서 중요한 요점이나 단어를 나열하는 방식
- **용도**: 정보를 간결하게 전달, 불필요한 사족 제거, 가독성 향상
- **표현법**: 키워드·개요 중심 서술, 글머리표·번호 표시
- **사용처**: 격식보다 실용성이 중요한 공적 관계 / 부가해설 없이 사실만 나열

### 3-2. 서술식 → 개조식 전환 예시 (강의자료 핵심 사례)

원문(서술식) — 강의자료 p.12 :

> \<중동항로와 관련된 특이사항\>
> 이슬람 최대 명절 중 하나인 라마단이 지난 8월 18일에 끝났습니다. 따라서 중동항로의 거래량과 실재 적재비율이 다시 늘어날 것으로 보입니다. (라마단 직전의 실재 적재비율은 95%에 육박했습니다.) 또한 중동 항로 선사협의체에서는 2012년 7월 중 컨테이너 당 300달러의 성수기 할증료를 부과할 예정이었으나 이를 유예했습니다.

개조식 변환:

```
〈중동항로 관련 이슈〉
 - 라마단(2012.7.20~2012.8.18) 종료로 중동항로 물동량 및 소석률 회복 예상
 - IRA가 7월 중 적용예정이던 PSS(USD300/TEU)를 유예함
```

**핵심 변화**:
1. 제목에서 `~와 관련된` 제거 → `~ 관련 이슈`
2. 한 덩어리 문단을 **2개 핵심 항목**으로 분할
3. 숫자·날짜를 **괄호로 묶어 강조**
4. `~할 것으로 보입니다` → `~ 예상`
5. `~할 예정이었으나 이를 유예하였습니다` → `~ 유예함`

## 4. 「적의를 보이는 것들」 — 빼야 할 4가지

업무 문장에서 가장 자주 군더더기가 되는 표현 4종.
Claude는 layout_optimizer.py 단계에서 이 4가지를 **자동 점검·경고**합니다.

### 4-1. 접미사 `-적`

> 빼도 의미가 유지되면 빼는 것이 원칙.

| 어색함 | 자연스러움 |
|--------|-----------|
| 사회적 현상 | 사회 현상 |
| 경제적 문제 | 경제 문제 |
| 정치적 세력 | 정치 세력 |
| 행정적 절차 | 행정 절차 |
| 조직적 대응 | 조직 대응 / 체계적 대응 |

다만 `정책적 판단`, `전략적 대응`처럼 **강조가 필요한 경우는 유지 가능**.

### 4-2. 조사 `의`

> `A의 B` → `A B` 또는 동사형 전환.

| 어색함 | 자연스러움 |
|--------|-----------|
| 매출의 감소 | 매출 감소 |
| 문제의 해결 | 문제 해결 |
| 업무의 효율성의 개선 | 업무 효율 개선 |
| 부모와의 화해 | 부모와 화해하는 일 |

특히 한 문장에 `의`가 2번 이상 나오면 **반드시 점검**.

### 4-3. 의존명사 `것`

> `~하는 것이 필요하다` → `~해야 한다` / `~가 필요하다`.

| 어색함 | 자연스러움 |
|--------|-----------|
| 검토하는 것이 필요합니다 | 검토가 필요합니다 |
| 개선하는 것이 중요합니다 | 개선이 중요합니다 |
| 증가할 것으로 예상됩니다 | 증가가 예상됩니다 |
| 문제가 발생한 것으로 확인됩니다 | 문제가 발생했습니다 |
| 대응 방안을 마련하는 것이 중요합니다 | 대응 방안 마련이 중요합니다 |

### 4-4. 복수 표현 `들`

> 이미 복수 의미가 있는 말 뒤의 `들`은 생략.

생략 후보 앞단어: 여러 / 많은 / 수많은 / 모든 / 각 / 대부분의 / 다양한

| 어색함 | 자연스러움 |
|--------|-----------|
| 여러 사람들 | 여러 사람 |
| 많은 직원들 | 많은 직원 |
| 각 부서들 | 각 부서 |
| 모든 매장들 | 모든 매장 |

## 5. 문장 줄이기 5원칙

### 5-1. 한 문장 한 핵심

> 한 문장에는 하나의 핵심만. 두 개 이상이면 분리.

❌ 라마단이 끝났기 때문에 중동항로의 거래량과 적재비율이 다시 늘어날 것으로 보이고, 라마단 직전 적재비율은 95%에 육박했으며, 선사협의체는 성수기 할증료를 부과할 예정이었으나 유예했습니다.

✅
- 라마단 종료 후 중동항로 거래량·적재율 회복 예상
- 라마단 직전 적재율은 95% 수준
- 성수기 할증료 부과는 유예

### 5-2. 한 문장 한 줄

> 어떤 문장이라도 1줄(약 35–45자)을 넘기면 분리하거나 요약.

페이지 가독성 점검의 1차 기준. 2줄 넘는 문장이 있으면 layout_optimizer.py가 경고.

### 5-3. 제목은 명사형으로 짧게

> 제목은 문장이 아니라 **검색어**처럼.

| 긴 제목 | 짧은 제목 |
|---------|-----------|
| 중동 항로와 관련된 특이사항 | 중동항로 특이사항 |
| 라마단 종료에 따른 중동항로 거래량 증가 가능성 | 라마단 종료 후 중동항로 전망 |
| 매장 운영의 효율성 제고 방안 | 매장 운영 효율화 방안 |
| 신규 사업의 추진의 필요성 | 신규사업 추진 필요성 |

### 5-4. 판단·예정 표현 압축

| 길게 | 짧게 |
|------|------|
| ~할 것으로 보입니다 | ~ 예상 / ~ 전망 |
| ~할 예정이었으나 이를 유예하였습니다 | ~ 예정 → 유예 |
| ~한 것으로 판단됩니다 | ~ 판단 / ~로 보임 |
| 중 하나인 | 대표적인 / 주요 / 생략 |
| 실제 적재비율 | 적재율 |
| ~와 관련된 | ~ 관련 / ~ / 생략 |
| ~에 대한 | ~ / 생략 |

### 5-5. 숫자는 눈에 띄게

업무 문서에서 숫자는 판단 근거. 문장 속에 숨기지 말고 다음 중 하나로:

- 항목 끝 우측 정렬 (`매출 5,210억원`)
- 별도 표로 분리
- 굵게/색상 강조

## 6. 두괄식 (보고서 핵심 원칙)

> **개요는 핵심을 첫 3줄 이내에 요약.**

특히 1페이지 보고서는 **음영 박스(요약문)** 안에 보고 목적을 1–2줄로 압축해 맨 앞에 둡니다.

예시 (공공기관 우수예시):

```
[음영 박스]
○○공사와 중소기업 상생협력을 위한 ○○기관과의 협의진행과정을 보고드림.
```

이 한 문장이 의사결정자가 **30초 안에 읽고 판단**하는 핵심.

## 7. 구성 점검: 표·이미지·도식의 활용

### 7-1. 표는 문장을 돋보이게 도와준다

> 표 그 자체가 본문이 아니다. 본문 문장이 핵심이고, 표는 근거.

좋은 표의 조건:
- 본문에서 말한 결론을 뒷받침
- 단위(`(단위: 천원)` 등) 우측 상단 표기
- 비고 열에 핵심 비교지표(`'19년/'16년 대비율 116%`) 배치
- 색상은 강조용으로만 (파랑·빨강 절제)

### 7-2. 글머리·블릿 위계

공공기관 사무관리지침 + 우수예시 기반 표준 위계:

```
□ 대제목 (16pt, 굵게)
  ○ 본문 (15pt)
    - 세부 (14pt)
      * 부가설명 (13pt, 회색)
        ※ 비고·주의사항 (13pt)
```

또는 행정적 번호체계:

```
Ⅰ. 추진배경
  1. 환경분석
    가. 외부환경
      (1) 거시환경
        (가) 정책 동향
          ① 세부 항목
```

이 두 체계를 **혼용하지 말 것**. 한 문서 안에서는 한 체계만.

### 7-3. 이미지·픽토그램

- 매뉴얼·안내문에는 픽토그램 적극 활용
- 보고서·시행문에는 절제 (표·간단한 도식 위주)

## 8. 양식의 중요성 (Why standardization matters)

> 공통 양식은 문서를 예쁘게 만드는 틀이 아니라, **조직이 같은 기준으로 일하게 만드는 소통의 약속**이다.
> — 사례강좌 0.양식의 중요성 中

양식이 통일되어 있을 때의 효과:
- 필요한 정보가 빠지지 않음
- 검토자가 빠르게 내용 파악
- 담당자가 바뀌어도 업무 흐름 이해 용이
- 다른 부서와 협업 시 설명 비용 감소
- 보고·승인 과정 명확

좋은 양식의 5조건:
1. **목적이 분명** — 왜 작성하는지 바로 알 수 있어야 함
2. **작성 순서가 자연스러움** — 배경 → 현황 → 문제 → 대안 → 요청
3. **중복 입력이 적음** — 같은 내용 두 번 쓰지 않음
4. **검토자가 보기 쉬움** — 작성자 편의보다 보고받는 사람의 이해 우선
5. **업무 기록으로 남기 좋음** — 검색·인수인계·감사·평가 활용 가능

## 9. 실무 체크리스트 (작성 후 점검용)

문서 빌드 전 Claude가 자동 점검하는 항목:

### 9-1. 표현 점검 (layout_optimizer 자동)
- [ ] `-적`을 빼도 자연스러운 표현 → 제거
- [ ] `의`가 한 문장에 2번 이상 → 분리·압축
- [ ] `것`, `것으로`, `하는 것` 반복 → 동사형으로
- [ ] `여러/많은/각/모든` 뒤에 `들` → 제거
- [ ] `~와 관련된`, `~에 대한` → 더 짧게
- [ ] 한 문장이 1줄(약 40자) 초과 → 분리

### 9-2. 구성 점검
- [ ] 제목이 명사형으로 압축되어 있는가
- [ ] 개요(요약문)가 3줄 이내 두괄식인가
- [ ] 추진배경 → 현황 → 해결방안 → 기대효과 흐름이 보이는가
- [ ] 빠진 필수 항목에 `※ AI 보완 필요` 플래그가 있는가
- [ ] 숫자·날짜·금액이 강조 정렬되어 있는가

### 9-3. 양식 점검
- [ ] 글머리 위계가 한 체계로 통일되어 있는가 (□○-* 또는 Ⅰ.1.가.(1))
- [ ] 한 문단이 페이지를 넘기지 않는가
